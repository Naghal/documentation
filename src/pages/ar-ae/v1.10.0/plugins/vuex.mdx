تتيح لك هذه الإضافة التقاط التغييرات/الحالة في VueX وفحصها لاحقًا أثناء إعادة تشغيل تسجيلات الجلسات. هذا مفيد جدًا لفهم وإصلاح المشاكل.

<Aside type="tip">
نحن نملك أيضًا درسًا مفصلًا حول كيفية استخدام إضافة VueX لالتقاط حالة من تطبيقك عند استخدام هذه المكتبة، يمكنك التحقق منه [هنا](/tutorials/vuex).
</Aside>

## التثبيت


```bash
npm i @openreplay/tracker-vuex
```


## الاستخدام

قم بتهيئة حزمة `@openreplay/tracker` كالمعتاد وقم بتحميل الإضافة إليها. ثم ضع الإضافة المنشأة في حقل الإضافات الخاص بمتجرك.

### إذا كان موقع الويب هو تطبيق واحد صفحة (SPA)


```js
import Vuex from 'vuex'
import OpenReplay from '@openreplay/tracker';
import trackerVuex from '@openreplay/tracker-vuex';
//...
const tracker = new OpenReplay({
  projectKey: PROJECT_KEY,
  // ... options 
});
```


### إذا كان التطبيق الإلكتروني منشورًا على الخادم (SSR)

اتبع المثال أدناه إذا كان التطبيق SSR. تأكد من أن `tracker.start()` يتم استدعاؤه مرة واحدة عند بدء التطبيق (في `useEffect` أو `componentDidMount`).
ارجع إلى قسم [الإصلاحات](troubleshooting/js-errors) إذا كنت تحصل على أخطاء مع SSR.


```js
import Vuex from 'vuex'
import OpenReplay from '@openreplay/tracker/cjs';
import trackerVuex from '@openreplay/tracker-vuex/cjs';
//...
const tracker = new OpenReplay({
  projectKey: PROJECT_KEY,
  // ... options
});
```


#### مع إصدار tracker-vuex أعلى من 4.0.0


```js
// check list of available options below
const vuexPlugin = tracker.use(trackerVuex(<options>)); 

// add a name to your store, optional (will be randomly generated otherwise)
const storeTracker = vuexPlugin('STORE NAME') 

const store = createStore({
  state,
  mutations,
  plugins: [storeTracker]
})
```


#### مع إصدار tracker-vuex أقل من 3.0.0


```js
// check list of available options below
const vuexPlugin = tracker.use(trackerVuex(<options>));  

tracker.start();

const store = new Vuex.Store({
  //...
  plugins: [vuexPlugin],
});
```


## الخيارات

يمكنك تخصيص سلوك الإضافة باستخدام الخيارات لتنظيف البيانات الخاصة بك. هذه مشابهة للخيارات الموجودة في الإضافة القياسية `createLogger`.


```js
trackerVuex({
  filter (mutation, state) {
    // returns `true` if a mutation should be logged
    // `mutation` is a `{ type, payload }`
    return mutation.type !== "aBlacklistedMutation";
  },
  transformer (state) {
    // transforms the state before logging it.
    // for example return only a specific sub-tree
    return state.subTree;
  },
  mutationTransformer (mutation) {
    // mutations are logged in the format of `{ type, payload }`
    // we can format it any way we want.
    return mutation.type;
  },
})
```
