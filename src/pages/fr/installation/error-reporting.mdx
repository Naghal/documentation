Le rapport d'erreur est très important pour surveiller la stabilité de votre application, repérer les régressions à l'avance et connaître les problèmes les plus fréquents rencontrés par vos utilisateurs. Par défaut, toute exception non capturée est enregistrée dans l'onglet «Erreurs»:

![Onglet Erreurs](/static/errors-tab.jpg#center)

De plus, les erreurs enregistrées apparaîtront également dans l'enregistrement de session sous l'onglet «Console» dans DevTools:

![Onglet Exceptions](/static/exceptions-tab-in-recording.jpg#center)

## Journalisation manuelle des exceptions

Le suivi OpenReplay prend en charge la journalisation de 3 types d'erreurs:
* Exceptions capturées - [Erreur](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Error)
* Promesses rejetées - [PromiseRejectionEvent](https://developer.mozilla.org/en-US/docs/Web/API/PromiseRejectionEvent)
* Événements d'erreur (onError) - [ErrorEvent](https://developer.mozilla.org/en-US/docs/Web/API/ErrorEvent)

### onError (ErrorEvent)

Cet exemple montrera comment signaler une erreur si un élément 
```js
tracker.handleError(error, metaObject); // metaObject is optional and is a flat object
```
 n'a pas été chargé:


```js
try {
    // application code
}
catch(err) {
    // application code that handles the error
    tracker.handleError(error, metaObject); // metaObject is optional and is a flat object
}
```

```js
function myFunc() {
    doSomeAsyncStuff()
    .then((result) => {
        // application code
    })
    .catch(promiseRejectionErr => {
        // application code to handle the error
        tracker.handleError(promiseRejectionErr, metaObject); // metaObject is optional and is a flat object     
    })
}
```
